pydsl provides verbs, actions that can be executed with any grammar or alphabet and some data

Check

Checks if an input conforms a Grammar or an Alphabet. Returns True or False.

>>> from pydsl.Config import load_default_memory
>>> load_default_memory()
>>> from pydsl.Check import check
>>> check("integer","abc")
False
>>> check("integer","1234")
True

Guess

Given an input and a repository with grammar and alphabets, it checks the input against every available definition, returning a list of valid grammars

>>> from pydsl.Config import load_default_memory
>>> load_default_memory()
>>> from pydsl.Guess import guess
>>> guess("1234")
{'cstring', 'hex', 'unixFilename', ..., 'ascii'}

Parser

Receives an input and generates an expanded tree known as a parse tree

Validate

Given an input that doesn't belong to a grammar, reports what needs to be fixed in the input to belong to that grammar

What's the pydsl approach?

It uses the expanded definition of the input through Parsers. A tree of valid invalid nodes is generates, at the moment you'll need to inspect the tree

>>> from pydsl.Config import load_default_memory
>>> load_default_memory()
>>> from pydsl.Validate import validate
>>> validate("Date","123")
[]
>>> validate("Date","123,3")
[]
>>> validate("Date","123/3")
[]
>>> validate("Date","123/3/13")
[]
>>> validate("Date","12/3/13")
[]

Translator

Translates an input to an output

Encoding conversion

In pydsl, encodings are described as alphabets

>>> from pydsl.Config import load_default_memory
>>> load_default_memory()
>>> from pydsl.Translator import translate
>>> translate("echo",{"input":"123"})
'123'

Diff

Shows the difference between two inputs that belong to the same alphabet
